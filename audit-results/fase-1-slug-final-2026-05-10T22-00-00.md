## 3. Arquivos com escopo diferente — IGNORADOS (justificados)

### 3.1 `src/utils/exporters/exportExcel.ts:14-21`

Função `slugify` para nome de arquivo Excel de download. Lógica diferente da canônica:
- `replace(/[^a-zA-Z0-9]+/g, '-')` (qualquer não-alfanumérico vira hífen, não só whitespace)
- `replace(/^-+|-+$/g, '')` (trim de hífens nas pontas)
- `toLowerCase()` no fim

Output: `slugify("Tamires Cássia D'Britto")` = `tamires-cassia-d-britto`. **Filenames preferem hífens** para legibilidade — não usar a canônica aqui.

### 3.2 `src/utils/exporters/exportPdf.ts:427-432`

Implementação inline equivalente para nome de arquivo PDF. Mesmo motivo do exportExcel.

## 4. Normalizações fuzzy — NÃO migradas (mapeadas e justificadas)

Estas 6 ocorrências normalizam nomes para **comparação fuzzy** (matching cross-source de cliente/colaborador), não geram identificadores estáveis. **Mantêm espaços** ou usam **UPPER** — não são alvo do Princípio 1.

| Arquivo : linha | Função | Por que NÃO é slug |
|---|---|---|
| `services/firebase.ts:376` | `renomearColaborador.normalize` | Colapsa whitespace para single space (mantém espaços), comparação de nome |
| `services/firebase.ts:840` | `corrigirNomeClientePoupanca.norm` | Lower + trim, sem remover caracteres especiais |
| `services/aumIntegration.ts:122` | inline | UPPER + trim, mantém espaços (chave de Map de comparação) |
| `utils/financials.custos.ts:181` | `calcularCustoDiretoCliente.normalize` | Lower + colapsa espaços (lookup cliente↔colaborador) |
| `state/AppContext.tsx:177` | `normNome` | UPPER + trim (detecção de Pure Asset cross-source) |
| `features/poupanca/usePoupanca.ts:906` | `norm` | Lower + trim (matching MM6 `clientes_base/` × `poupanca/`) |

Não substituir essas por `slug()` quebraria o casamento — `slug("João Silva") = "joao_silva"` mas a comparação precisa de `"joão silva"` (com espaço, lowercase) ou `"JOÃO SILVA"` (UPPER) para casar com strings que vêm de fontes diferentes.
